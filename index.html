Actuals — Weekly PM Update

Enter actual days worked per named role. Forecast-copied names can be updated in Actuals without changing the locked Forecast baseline.

Values can be decimals such as 0.5, 1, 2.5 or 5. Negative values are blocked. Any value above 7 days in a week is highlighted as a warning. Name is editable for every Actuals row. Added rows are actual-only and do not modify the locked contract Forecast.

Customer-safe export rule

Customer-safe PDFs aggregate supporting detail by stream / role / code and hide individual names and day rates. Use Internal detail only for internal review.

Invoice Amount (Net) is calculated for the selected project code as actual days in the invoice period multiplied by each resource day rate. Variance notes compare actual days to the locked Forecast for the same code and period.
